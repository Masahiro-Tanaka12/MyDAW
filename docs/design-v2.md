# 設計計画書 v2.0 — First Song 進化版

作成日: 2026-07-02

---

## 現状の問題

v0.1.0 の操作は「ムードを選んでボタンを押す」だけ。  
Tone.js のシンセサイザー音のみ。ユーザーの関与がほぼゼロ。  
「曲を作った」という感覚が生まれにくい。

---

## 新しいビジョン

> **「入口を絞り、出口を広げる」ハイブリッドDAW**

| フェーズ | モード | ユーザー |
|---|---|---|
| v0.1.0 | 全自動（ムード → すべて決まる） | 超初心者 |
| v2.0 直感モード | 半自動（カード選択 → 高音質で鳴る） | 初心者 |
| v2.0 DAWモード | 直感モードの土台を細かく編集 | ステップアップ後 |

「作った感」は選択の数に比例する。選択肢は絞り込み、迷わせない。  
慣れたユーザーはDAWモードに移行して仕上げができる。

---

## コンセプトの変化

| 項目 | v0.1.0 | v2.0 |
|---|---|---|
| ユーザーの操作 | ムード選択のみ | ムード → コード選択 → 楽器選択 |
| 音源 | Tone.js シンセ（合成音） | 実楽器サンプル音 |
| コード | 自動決定 | 候補から選ぶ |
| ギター・ベース | シンセで代替 | 実音ファイル使用 |
| 打ち込み | なし | ステップ選択式（ピアノロールではない） |

## 主要機能設計

### 1. コード選択 UI（ハイブリッド設計）

#### ステップ1: 4コードセットをカードで選ぶ（初期配置）

ムード選択後、雰囲気の名前がついた「4コードセット」カードを 3〜4 枚提示する。

```
[ 王道ポップス    ]  Am → F → C → G
[ チル・Lo-Fi    ]  Dm → Am → Bb → F
[ 切ない系       ]  Am → Em → F → C
[ スパニッシュ   ]  Am → G → F → E
```

ユーザーは 1 枚タップするだけで 4〜8 小節の土台が完成する。

#### ステップ2: 1コードずつ差し替える（フロー型への移行）

配置されたコードのうち 1 つをタップすると、その前後の流れに合う「差し替え候補」が出る。

```
現在: Am → F → [C] → G
          ↓ Cをタップ
差し替え候補:
  ● Am（おすすめ）
  ● Em
  ● Dm
```

候補はダイアトニックコード理論に基づいてデータベースから抽出する。AI は使わない。

**プロトタイプ実装順: ステップ1（カード選択）から先に作る。**  
ステップ2（差し替え）は土台完成後に追加する。

---

### 2. ギター音源

**方針：Tone.js Sampler + 実音サンプルファイル**

- コードを「コードシェイプ単位」でサンプリングする
- Am, F, C, G, Em, Dm, E, A など使用頻度の高い 12〜16 コードをカバー
- ストロークパターンは別途選択（↓↑パターンを数種類）

```
samples/guitar/
  Am.wav   F.wav   C.wav   G.wav
  Em.wav   Dm.wav  E.wav   A.wav
  ...
```

ストロークタイミングは Tone.js の Transport で制御。  
実音を小さなスケジュールでトリガーしてストロークを再現する。

---

### 3. ベース音源

**方針：Tone.js Sampler + 単音サンプル**

- ベースはコードルートに連動した単音を使う
- 音域: A1〜G3 程度をサンプリング
- パターンは「ルート刻み」「歩くベース」「シンプルなリフ」から選択

```
samples/bass/
  A1.wav  B1.wav  C2.wav  ... G3.wav
```

v0.1.0 の「固定音程ベース」問題をここで根本解決する。  
コードが変わるとベースルートが自動追従する。

---

### 4. ドラム音源

**方針：Tone.js Sampler + ドラムサンプル**

```
samples/drums/
  kick.wav   snare.wav   hihat_closed.wav   hihat_open.wav
  rimshot.wav   crash.wav
```

v0.1.0 の「キックのみ」問題を解決。  
ムードごとにパターンを切り替える。

---

### 5. ステップ選択式 メロディ入力（直感モード）

直感モードではピアノロールを使わない。  
代わりに **スケール内の音を番号で選ぶ** インターフェースを提供する。

```
[ ド ] [ レ ] [ ミ ] [ ファ ] [ ソ ] [ ラ ] [ シ ]
  1      2      3      4       5      6      7
```

選んだ音が並ぶだけ。タイミングは自動補正（クオンタイズ）。  
「作った」という感覚を生みながら、音痴にならない設計。

### 6. DAWモード（ステップアップ後）

直感モードで作った楽曲データを引き継ぎ、以下を提供する。

- **ピアノロール**: 直感モードで生成されたノートを可視化・編集
- **トラックミキサー**: 音量・パン・ミュート・ソロ
- **Smart FX Wizard**: 「ズッシリした低音」などの感覚ラベルで EQ/コンプを調整
- **エクスポート**: WAV / MP3 書き出し

*DAWモードは Phase 3 以降に実装する。直感モードが完成してから着手する。*

## UI フロー（v2.0）

```
① ムード選択
   ↓
② コード進行を選ぶ（3〜4候補）
   ↓
③ ギタースタイルを選ぶ（アルペジオ / ストローク / なし）
   ↓
④ ベーススタイルを選ぶ（シンプル / 歩く / なし）
   ↓
⑤ メロディを選ぶ（なし / 候補A / 候補B / 自分で並べる）
   ↓
⑥ 再生 → 気に入ったら保存
```

「なし」を選べば v0.1.0 と同じ体験になる。  
ステップをスキップする「おまかせ」ボタンも用意する。

## 技術設計

### サンプル音源の取り込み

Electron なので Node.js の `fs` でローカルファイルを読める。  
ただし Tone.js Sampler は URL または ArrayBuffer を受け取る。

**方針A: プロジェクトにバンドル（推奨）**
- `public/samples/` に wav を配置
- Vite のアセット扱いで URL 参照できる
- 配布サイズが増えるが、オフライン動作が確実

**方針B: ユーザーがサンプルを持ち込む**
- Electron の `dialog.showOpenDialog` でファイル選択
- ユーザーが好きな音を使える
- 複雑なため Phase 4 以降で検討

**方針C: ネット上の無料サンプルライブラリを使用（採用）**
- Freesound.org（CC0 / CC BY）や VSCO 2 Community Edition などから調達
- プロジェクトに `public/samples/` としてバンドル
- オフライン動作が確実で、ライセンスも明確

→ **Phase A は方針Cで進める**

---

### Tone.js Sampler の使い方

```typescript
const guitarSampler = new Tone.Sampler({
  urls: {
    'A2': 'samples/guitar/Am.wav',
    'F2': 'samples/guitar/F.wav',
    // ...
  },
  baseUrl: '/',
}).toDestination()

// 再生時
guitarSampler.triggerAttackRelease('A2', '2n', Tone.now())
```

コードごとに対応するサンプルキーをマッピングするテーブルを作る。

### ストロークパターンの実装

```
小節 1    |  ↓  ↑  ↓  ↑  |
タイミング  |  0  .5  1  1.5 | (拍)
```

各ストローク = `triggerAttackRelease` の呼び出し。  
ベロシティを変化させて強弱をつける。

---

### アーキテクチャ変更点

```
現在:
ComposerEngine → SongBlueprint → ChordPlayer(Synth)
                               → BassPlayer(Synth)
                               → DrumPlayer(Synth)

v2.0:
ComposerEngine → SongBlueprint → ChordPlayer(Sampler)
                               → BassPlayer(Sampler)
                               → DrumPlayer(Sampler)
                               → MelodyPlayer(Sampler or Synth)

SongBlueprint に「ユーザー選択」フィールドを追加:
  - selectedProgressionId
  - guitarStyle: 'arpeggio' | 'strum' | 'none'
  - bassStyle: 'simple' | 'walking' | 'none'
  - melodyPatternId: string | 'none'
```

## サンプル音源の調達方針

**使える無料音源（ライセンス確認済みのもの）**

- [Freesound.org](https://freesound.org) — CC0 または CC BY
- [SampleSwap](https://sampleswap.org)
- [Versilian Studios](https://vis.versilstudios.com/vsco-community.html) — VSCO 2 Community Edition (CC0)
- 自作録音（最も自由度が高い）

各サンプルは 44.1kHz / 16bit WAV で統一する。  
ループポイントは不要（Tone.js が制御するため）。

---

## 制約・考慮事項

| 項目 | 内容 |
|---|---|
| バンドルサイズ | サンプルを増やすとアプリが重くなる。1ファイル最大 300KB 目安 |
| 読み込み時間 | Sampler の初期化は非同期。ローディング画面が必要 |
| ライセンス | 使用する全サンプルのライセンスを docs に記録する |
| ピアノロール | **実装しない**（CLAUDE.md の禁止事項を維持） |
| VST | **実装しない** |
| AI生成 | **実装しない** |
