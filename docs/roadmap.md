# ロードマップ

## 判断基準（全フェーズ共通）

> 「音楽経験ゼロのユーザーが、この機能を見て迷わないか？」
> YES なら実装。NO なら実装しない。

---

## 直感モードの主軸（設計方針）

直感モードは **コード → ドラム → 音質 → メロディ** の4層の逐次選択が主軸。
ムード／ジャンルは生成のトリガーではなく、各層の候補を絞り込む任意のフィルタ（詳細: [architecture.md](architecture.md)）。

## v0.3.0 — DAWモード導入（CLAUDE.md Phase3相当）

**テーマ：直感モードで作った曲を、さらに作り込めるようにする**

直感モードとDAWモードは同じ `SongBlueprint` を共有する（CLAUDE.md「コンセプト」参照）。この版でDAWモードへの切り替えを初めて解放する。

### ピアノロール
- [ ] `SongBlueprint.tracks` をノート（横軸：時間、縦軸：音高のブロック）として可視化
- [ ] ノートの長さ・位置・ベロシティの編集
- [ ] **（構想）編集時の説明機能**：ノートやコードを触ると「なぜこの音が鳴っているか」を軽く表示する
  - 例：選んだノートがコードのroot/3rd/5thのどれか、ダイアトニックコード内かどうかをツールチップ的に出す
  - v1.0.0（初心者教育）の解説パネル構想を、ピアノロールでの実操作の文脈に前倒しで組み込むイメージ。常時表示はせず、任意でON/OFFできる補助情報とする
  - 「音楽経験ゼロのユーザーが迷わないか」ではなく「ステップアップ後のユーザーが理論を学べるか」が判断基準（DAWモードの開発ルールに従う）

### トラックミキサー
- [ ] 音量・パン・ミュート・ソロの手動調整
- [ ] 直感モードの自動ミックス値（EQ/コンプ/リバーブ/パン）を初期値として引き継ぐ

## 直感モードには実装しない機能

| 機能 | 理由 |
|---|---|
| VST / プラグイン管理 | 初心者には不要、設定コストが高い |
| MIDI 詳細編集 | 専門知識が必要 |
| ピアノロール | 専門的すぎる。v0.3.0（DAWモード導入）で実装する |
| 複数トラックの手動ミキサー | 自動で十分。v0.3.0（DAWモード導入）で実装する |
| テンポ変動（ritardando 等）| 初心者には高度 |

## 全バージョン共通の禁止事項

| 機能 | 理由 |
|---|---|
| AI による曲生成 | データベース駆動の方針を維持する |
