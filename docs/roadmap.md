# 開発ロードマップ (roadmap.md)

## 判断基準（全フェーズ共通）

> 「音楽経験ゼロのユーザーが、この機能を見て迷わないか？」
> YESなら実装。NOなら実装しない。

---

## Phase 0 ── 基盤構築

**目標**: 開発環境を整える。まだ音は鳴らない。

### タスク

- [ ] Electron + Vite + React + TypeScript プロジェクト初期化
- [ ] Tone.js インストールと疎通確認（コンソールで音が鳴ること）
- [ ] Zustand セットアップ
- [ ] Tailwind CSS セットアップ
- [ ] `data/` フォルダに音楽理論JSONの雛形を作成
- [ ] フォルダ構成を `architecture.md` に沿って整備

### 完了条件

`npm run dev` で Electron ウィンドウが開き、空白の画面が表示される。

---

## Phase 1 ── 音を鳴らす（MVP）

**目標**: ユーザーが5ステップで音楽を再生できる。完成した曲は聴けるだけでよい。

**期間目安**: 2〜4週間

### 実装内容

#### UIウィザード（6画面）
- [ ] WelcomeScreen: 「はじめる」ボタンのみ
- [ ] GenreScreen: ジャンル選択（5カード）
- [ ] MoodScreen: 雰囲気選択（2〜3カード、ジャンルに応じて変化）
- [ ] ChordScreen: コード進行選択（3カード＋試し聴きボタン）
- [ ] MelodyScreen: メロディ選択（3カード＋試し聴きボタン）
- [ ] PlayScreen: 再生・停止ボタン。ループ再生。

#### 音楽理論レイヤー
- [ ] genres.json の実装（5ジャンル）
- [ ] chord-progressions.json の実装（最低8進行）
- [ ] melody-patterns.json の実装（最低3パターン）
- [ ] db.ts: ジャンル＋ムードでコード進行・メロディ候補をフィルタする関数

#### 音声エンジン
- [ ] arranger.ts: ChordProgressionとMelodyPatternからScheduledNote[]を生成
- [ ] engine.ts: Tone.jsで再生・停止・BPM設定
- [ ] 最低限の自動ミックス（各トラックの音量バランスのみ）

#### Phase 1 で意図的に省略するもの
- 曲の保存・エクスポート
- 曲構成（イントロ/バース/コーラス）→ ループ再生のみ
- 楽器の変更
- BPMの手動調整
- ドラムトラック（シンプルなリズムのみ、または省略）

### 完了条件

音楽経験ゼロの人が、5ステップで「それっぽい音楽」を再生できる。

---

## Phase 2 ── 曲として成立させる

**目標**: ループではなく「1曲」の形になる。イントロ→バース→コーラス→アウトロ。

**期間目安**: 3〜5週間

### 実装内容

#### 曲構成の自動生成
- [ ] Song構造: イントロ(4小節) → バース(8小節) → コーラス(8小節) → アウトロ(4小節)
- [ ] 各セクションで異なるベロシティ・アレンジを適用（コーラスは盛り上がる）

#### ドラムトラック
- [ ] ジャンル別ドラムパターンのJSON化
- [ ] drumKit音源の選定（Tone.js標準 or サンプル）

#### ベーストラック
- [ ] コードのルート音を8分音符で鳴らすシンプルなベース
- [ ] ジャンルに応じたベースラインの変化（pop: 4分音符, rock: 8分音符）

#### コード伴奏トラック
- [ ] ジャンル別ボイシングパターン（pop: 4分音符刻み, lofi: 2小節に1回）

#### CompleteScreen（完成画面）
- [ ] 曲にタイトルをつける入力欄
- [ ] 「完成！」の演出
- [ ] もう一度作るボタン

### 完了条件

曲を最後まで再生したとき、「1曲完成した」と感じられる。

---

## Phase 3 ── 音を良くする

**目標**: 自動ミックスの品質を上げ、聴いて満足できる音質にする。

**期間目安**: 2〜4週間

### 実装内容

#### 自動ミックス強化
- [ ] EQ自動設定（各パートの周波数帯を整理）
- [ ] コンプレッサー適用（音量のばらつきを抑える）
- [ ] リバーブ量のジャンル別自動調整
- [ ] マスターリミッター（音割れ防止）
- [ ] パン自動設定（メロディ=センター, コード=少し広げる）

#### 音源の追加
- [ ] ピアノ、ギター、シンセ、8bitの最低4種類の楽器
- [ ] 楽器選択画面（PlayScreen内、オプションとして）

#### エクスポート機能
- [ ] WAVファイルとして書き出す（Electron + OfflineAudioContext）

### 完了条件

出力したWAVを人に聴かせて「自分で作ったBGM」として恥ずかしくない品質。

---

## Phase 4 ── 初心者教育

**目標**: 「なぜこの曲が良いのか」を、音楽理論を知らない人にも伝える。

**期間目安**: 3〜5週間

### 実装内容

#### コード進行の説明
- [ ] ChordScreen で「このコード進行を選ぶと○○な感じになる理由」を表示
- [ ] 有名曲の例を添える（例: 「王道進行は『残酷な天使のテーゼ』などで使われています」）

#### 音楽理論ミニ解説
- [ ] PlayScreen に「今流れているのは○○スケールです」などの情報を表示（折りたたみ式）
- [ ] ノート/ダイアリー機能：「今日学んだこと」を記録できる

#### 段階的な開放
- [ ] Phase1〜3でUIを固定してきた設定（BPM, キー）を手動変更できるようにする
- [ ] 「もっと詳しく設定する」モードを追加（上級者オプション、デフォルトOFF）

### 完了条件

このソフトで5曲作ったユーザーが、「コードって何か少しわかった」と言える。

---

## フェーズ別 実装しない機能

| 機能 | 理由 |
|---|---|
| VST/プラグイン管理 | 初心者には不要、設定コストが高い |
| MIDIファイルのインポート/エクスポート | 専門知識が必要 |
| ピアノロール（音符の直接編集） | 専門的すぎる、Phaseを超えてから検討 |
| 複数トラックの手動ミキサー | 自動で十分 |
| テンポ変動（ritardando等） | 初心者には高度 |
| 複数曲の管理 | Phase1〜2では1曲の完成に集中 |
