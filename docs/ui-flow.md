# UI設計・画面遷移書 (ui-flow.md)

## 設計原則

- ユーザーが次に何をすべきか、**常に1つだけ**明確にする
- 「設定」は一切ない。あるのは「選択」だけ
- 「コード進行」という言葉は画面に一切出さない
- 迷ったら「初心者はこれを理解できるか？」で判断する

---

## 全体フロー

```
[1. ようこそ]
  ├── おまかせ → [5. 再生]（全自動・約5秒）→ [6. 完成！]
  │
  └── 自分で選ぶ
        ↓
      [2. ジャンル選択]
        ↓
      [3. 雰囲気選択]
        ↓
      [4. シーン選択]（コード進行は内部で自動決定）
        ↓
      [4b. メロディ選択]
        ↓
      [5. 再生]
        ↓
      [6. 完成！]
```

音楽経験ゼロでも、**「おまかせ」なら30秒で音楽が流れる**ことを目標とする。

---

## 各画面の詳細

---

### Screen 1: ようこそ（WelcomeScreen）

```
┌─────────────────────────────────────┐
│                                     │
│          🎵 First Song              │
│                                     │
│    音楽経験ゼロでも                   │
│    自分だけの1曲を作れる              │
│                                     │
│  ┌───────────────────────────────┐  │
│  │  ✨  おまかせで1曲作る          │  │
│  │      全自動・約5秒              │  │
│  └───────────────────────────────┘  │
│                                     │
│      [ 自分で選んで作る →  ]         │
│                                     │
│  ※ 自分で選ぶ場合: 約15〜30分        │
│                                     │
└─────────────────────────────────────┘
```

**ユーザーの操作**:
- **「おまかせで1曲作る」** → RecommendationEngineが全自動で曲を生成 → Play画面へ直行
- **「自分で選んで作る」** → 通常のウィザードフローへ

**システムの動作（おまかせ時）**:
`genre=pop, mood=happy, scene=元気` をデフォルト値として使い、
RecommendationEngineが最高スコアのコード進行・メロディ・BPMを決定する。

**ポイント**:
- 「おまかせ」を上に置く。迷う初心者がまず押すボタン
- 「自分で選ぶ」は控えめに表示。意欲のある人向けの選択肢

- 全4トラックを同期して再生（melody / chord / bass / drum）
- 自動ミックス適用済み（ユーザーには見えない）

**ポイント**:
- 選択サマリーはジャンル・シーン・メロディ名のみ（音楽用語なし）
- 音量スライダー・BPM調整などは一切表示しない

## 画面遷移まとめ

```
Welcome
  ├─→ [おまかせ] ────────────────────────────→ Play（全自動）
  │                                                 └─→ Complete
  │                                                       └─→ Welcome
  └─→ [自分で選ぶ]
          └─→ Genre選択
                └─→ Mood選択
                      └─→ Scene選択（コード進行は内部で自動決定）
                            └─→ Melody選択
                                  └─→ Play（再生確認）
                                        ├─→ Complete（完成）
                                        │       └─→ Welcome（もう1曲）
                                        └─→ [戻る] → Melody選択
```

- 各ステップから「前の画面に戻る」ことは常に可能
- 「スキップ」や「後で決める」は存在しない
- コード進行選択という概念はどの画面にも存在しない

## UIコンポーネント共通ルール

### 選択カード（SelectCard）

```
┌──────────────────────────────────┐
│ [アイコンまたは絵文字]             │
│                                  │
│ タイトル（10文字以内）             │
│ 説明（20〜30文字、感情ベースの言葉）│
│                      [▶ 試聴]   │
└──────────────────────────────────┘
```

- **選択済み**: 枠が強調色になる
- **試聴ボタン**: メロディ画面のみ表示（シーン選択では不要）

### 進捗バー（ProgressBar）

- 全5ステップを「●●●○○」形式で表示
- おまかせモードでは進捗バーを表示しない

### 共通ルール

- フォントサイズ: 最小16px
- 専門用語禁止: コード名（C, G, Am）、BPM、EQ、コンプなどは画面に出さない
- 確認ダイアログ禁止: 戻ったら素直に戻る

---

## ユーザーが絶対に見ないもの（内部処理）

以下はUIに一切表示しない：

- コード進行の名前・種類
- BPMの数値
- キー（C, Am など）
- コード名（C, G, Am, F）
- EQ, コンプ, リバーブのパラメーター
- 各トラックの音量数値
- スケール名（メジャー/マイナー）
- RecommendationEngineのスコア
