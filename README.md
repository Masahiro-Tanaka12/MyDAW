# First Song

> 音楽経験ゼロでも、30分以内に人生初の1曲を完成できる DAW。

---

## コンセプト

既存のDAWは「自由すぎる」。  
初心者が迷うのは、選択肢が多すぎるから。

**First Song** はあえて選べることを減らし、完成率を上げる。

直感モードの主軸は **コード → ドラム → 音質 → メロディ** の4層の逐次選択。  
ホーム画面はコード選択画面そのもので、ムード／ジャンルは生成のトリガーではなく各層の候補を絞り込む **任意のフィルタ**。  
テンポのみ例外的に数値スライダーで選べる。楽器・ミックスは自動で決まる。

> v0.2.0時点で、コード・ドラム・音質の3層は選択UIまで実装済み（詳細は下表）。メロディ層のみ未着手（鼻歌を録音し既存ライブラリで自動採譜する方式で実装予定。ピッチ検出は自前実装しない）。

## アーキテクチャ概要

```
[ユーザー] ── ムード（任意フィルタ）＋テンポ ──▶ [ComposerEngine]
[ユーザー] ── コード選択（テンプレート／1コードずつ） ──▶  │
[ユーザー] ── ドラム・音質選択 ──▶                          │
                                                  ▼
                                          [Selector] 候補を重み付き抽選
                                                  │ (SelectionSet)
                                                  ▼
                                          [Generator] 純粋関数
                                    │ chordTheory で度数→実音、ベース/ドラムをタイリング
                                                  ▼
                                          [SongBlueprint]
                                                  │
                                                  ▼
                                  [PlaybackEngine / Tone.js]
                                    │ masterVolume → Compressor → Limiter → Out
                                    │ 共有リバーブバス（Convolver）
                                    ├─ ChordPlayer（EQ3・パン・リバーブセンド）
                                    ├─ BassPlayer（同上）
                                    └─ DrumPlayer（kick/snare/hihat、EQ3等はまとめて適用）
```

`ComposerEngine`（オーケストレーター）・`Selector`（重み付き抽選）・`Generator`（`SelectionSet` + `ProgressionTemplate` → `SongBlueprint` の純粋関数）の3分割構成（`src/renderer/src/composer/`）。コード進行は`theory/chordTheory.ts`の度数ベース理論エンジンで解決し、キーに依存しない。

音楽理論のデータベースは TypeScript の Record オブジェクト（`src/renderer/src/data/music/`）として定義され、これが正式なデータ形式（JSONへの移行はしない）。AI は使用しない。

詳細は [アーキテクチャ詳細](docs/architecture.md) を参照。
