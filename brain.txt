アプリケーション開発設計書（プロトタイプ用）
1. アプリケーションのコアコンセプト
「音楽理論を視覚的な『正解の選択肢』に変換し、知識ゼロでも直感的にプロレベルの楽曲が作れるハイブリッドDAW」

従来のDAWのように「白紙の状態から音を打ち込む」のではなく、アプリ側がコード、ドラム、音響エフェクトの「相性の良い選択肢（正解）」を視覚的に提示し、ユーザーはそれを選択・配置していくことで破綻のない楽曲を構築できる。かつ、中〜上級者向けに、従来のDAWのような細かい打ち込みや微調整（MIDI編集、パラメーター操作、書き出し）ができる画面も共存させる。

2. 画面構成と主要機能（UI/UX）
① 楽曲構築画面（直感・ビジュアルモード）
音楽の知識がなくても、視覚的なナビゲーションに従ってパーツを選んでいくメイン画面です。

コード進行の視覚的ナビゲーション (Chord Progression Guide)

仕様: 最初のコード（和音）を選ぶと、音楽理論（コード進行のセオリー）に基づいて「次に繋がると気持ちいいコード（正解の選択肢）」が、おすすめ順に色分けやサイズ違いのボタンで視覚的に提示される。

AIへの指示用ワード: 音楽理論（ダイアトニックコード/ドミナントモーション）の視覚的サジェスト機能

ドラムパターンのセレクト機能 (Drum Pattern Selector)

仕様: ジャンル（Pop, Rock, Lo-Fiなど）や、選んだコードのテンポ（速さ）に自動で同期するドラムの骨組み（ループパターン）を、視覚的なカードUIから選択できる。

音作り・エフェクトの「正解」プリセット (Smart FX Wizard)

仕様: コンプレッサーやイコライザー（EQ）の難しい数値をいじる必要はない。「音が前に出る」「スッキリさせる」「低音をズッシリ」といった人間の感覚的な言葉（ラベル）を選び、スライダーで強弱を調整するだけで、内部のEQ/コンプのパラメーターが最適値に自動設定される。

② 伝統的なDAW画面（詳細編集・エディットモード）
直感モードで作った土台を、さらに細かく作り込みたい人のための画面です。トグルスイッチ等でシームレスに切り替えられます。

MIDIタイムライン＆ピアノロール (Piano Roll)

選択したコードやメロディが、従来のDAWのような「横軸：時間、縦軸：音高」のノート（ブロック）として可視化され、1音ずつ長さを変えたり、位置をずらしたりできる。

トラックミキサー (Track Mixer)

音量（ボリューム）、左右の定位（パン）、ミュート、ソロを細かく管理できる。

オーディオエクスポート (Audio Export)

完成した楽曲を WAV や MP3 などの音声ファイルとして書き出す機能。

3. システム開発のための技術的キーワード
Claude Codeに具体的な実装を頼む際、以下の技術スタックやアプローチを指定するとスムーズです。

フロントエンド / 音声処理: Webアプリとして作る場合、ブラウザ上でシンセサイザーの音を鳴らしたりエフェクトをかけたりするために Web Audio API や、それを扱いやすくしたライブラリ（Tone.js など）を使用する。

データ構造: 楽曲は「タイムライン（時間軸）に、どのコードやドラムパターンが配置されているか」を管理する JSON 形式のデータ構造（ステート管理）にする。
